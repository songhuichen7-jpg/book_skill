# 章节写作契约（给写章的 agent 的硬约束）

每章正文是一个 Typst 源文件 `$PROJECT/chapters/NN.typ`（NN 两位，如 `00.typ`、`01.typ`…）。**只写正文**——封面、扉页、章大标题由书框架自动加，章文件里不要写这些。

风格无关：下列组件由当前风格的 `template.typ` 导出，**任何风格都同名同签名**，所以正文在任何风格下都能原样编译。组件的视觉实现见 `$PROJECT/styles/<style>/template.typ`。

## 文件骨架

```typst
#import "../template.typ": *

#prose[
  这一章只回答一个问题：*<本章核心问题>*。

  == N.1　小节标题（id 与文字用全角空格）
  正文段落。大白话 + 类比讲概念，*重点加粗*，`行内代码`。
  - 列表项
]

#keypoint[ 一句话能记住的要点。 ]

#prose[
  == N.2　另一个小节
  引出代码的话……
]

```python
# 真实可运行的代码；模型名/标识符用本书统一约定的那一套
# 每行 ≤ ~74 字符，避免在版心里折行
```

#flow(("步骤A", "步骤B", "步骤C"), loopback: "可选：循环说明")

#prose[
  == N.3　……
  交叉引用写"见第 X 章"。
]

#jd-box(role: "岗位/场景名")[ 本章对应的真实能力点 / 小结。 ]
```

## 排版规则（违反会编译失败或难看）

- **第一行必须**是 `#import "../template.typ": *`。
- **叙述正文一律包在 `#prose[ ... ]` 里**（双栏）。`== 二级标题`（小节）、`=== 三级标题`、列表、`*加粗*`、`` `行内代码` `` 都放在 prose 内部。
- **跨整页宽的元素放在 prose 块「之间」**（不要塞进 prose 内）：代码块、`#flow(...)`、`#keypoint[...]`、`#jd-box(role:...)[...]`、`#note[...]`、`#table(...)`。
- `==` / `===` 即使缩进在 prose 内部也会被识别为标题（无需顶格）。
- **禁止**：`import` 外部包；`#set page` / `#page`；重定义样式；emoji；在 prose 外写裸段落文字。
- 长章别只用 `==`——在自然断点补几个 `===` 三级小节，阅读节奏更稳。

## 组件速查

| 组件 | 用法 | 说明 |
|---|---|---|
| `#prose[...]` | 包裹叙述 | 双栏正文容器 |
| 代码块 | ` ```python … ``` ` | 自动语法高亮 + 强调色左边线；跨整页宽 |
| `#keypoint[...]` | 要点 | 强调色"一句话记住"框 |
| `#jd-box(role:"…")[...]` | 章末小结/能力点（**仅求职/教材类书**） | 近黑"对应岗位要求 · …"框 |
| `#note[...]` | 注意/陷阱 | 灰色提示框 |
| `#flow((a,b,c), loopback:"…")` | 流程/步骤 | 编辑式数字圆 + 细连线 + 回环药丸 |
| `#loop-diagram()` | 示例循环 | flow 的一个特例，可不用 |
| `#diagram("name")` | **有结构的富表达图**（状态机/分组/带标签或虚线的边/反馈回路/多块） | 走 Mermaid（`mmdc`）渲染成 PNG 再嵌；详见 `$SKILL/references/diagrams.md`。线性步骤用 `flow()`，一出现分组/状态/回路就用它 |
| `#table(columns:(…), table.header[…], …)` | 对照表 | 全局已设为编辑式（无竖线、强调色表头、发丝行线）；**不要再传 stroke/inset**，交给全局规则统一 |

## 口味（教材/技术书）

- 通俗易懂：概念大白话 + 类比；代码硬核可运行；诚实讲权衡。
- 每章：开头点出**核心问题**；至少 1 个 `keypoint`；结尾一个收尾框；合适处用 `flow`/`table`/`note`/代码。
- **收尾框按书的类型选**：求职/能力向教材用 `jd-box`（标签是"对应岗位要求"）；其它书（实战手册、科普、指南）`jd-box` 的标签不合身，改用 `#keypoint[ 本章行动要点：… ]` 收尾、用 `#note` 放风险/冷水。（经验：在不对口的书里硬塞 `jd-box` 会读着别扭。）
- 引用事实/数字/项目时尽量核实（必要时用你所在环境的联网检索/抓取能力）；**闭源系统内部实现细节用「据社区逆向/分析」措辞**，别当官方事实。
