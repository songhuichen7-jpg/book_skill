# 风格 · 插画信息图（手账涂鸦 / sketchnote 风，纯白底 + 适度上色）

当一张图更适合做成**有趣、精致的插画信息图**（而非功能化流程图）时，用一个**图像生成模型**生成。把下面这段作为每张图 prompt 的**风格前缀**，再拼上该图的「内容 + 必须准确出现的标注 + 逻辑」。

`scripts/gen_infographics.py` 会逐张读取 `$PROJECT/diagrams/<name>.prompt.txt`，顺序调图像模型生成，存到 `$PROJECT/assets/diagrams/<name>.png`，并自动把近白背景**漂成纯白 #FFFFFF**。

## 风格前缀（原样放在每个 prompt 开头）

```
画一张「手账涂鸦 / sketchnote」风格的横版【彩色】信息图。视觉：纯白背景上用黑色细钢笔手绘的涂鸦——线条带一点轻松的手抖感、不死板；手绘的方框、圆圈、箭头、随手画的 doodle 小图标。
【上色——这是重点，必须明显有彩色】给主要方框/分区/节点/图标【填上】柔和色块（像马克笔/彩色便利贴平涂），每张至少 4 种不同颜色，选自：赤陶橙 #D97757、鼠尾草绿 #609A81、长春花紫 #827DBD、赭黄 #C2913B、灰蓝 #5E7E96、砖玫红 #C2766A。不同阶段/分支/分组用不同色区分；绝不要整张只有橙色、也绝不要只有黑白。黑色只用于线条和文字。整体 playful、清爽、不幼稚不卡通。
【背景铁律】整张图背景必须是纯白色 #FFFFFF，绝不要纸纹、米色、牛皮纸或任何底纹。
【文字铁律，最重要】所有文字标签必须清晰、工整、准确、无错字无乱码（用干净的手写体或印刷体都行），尤其中文务必正确——这是信息图，标签错一个字就废了。只渲染我明确给出的文字，不要自造、不要加多余的英文或装饰文字。
```

## 每张图 prompt 的写法（风格前缀之后接）

- 一句话说清这张图要让读者**一眼看懂什么**。
- 给一个**贴切又有趣的视觉隐喻/场景**（冰山、漏斗、阶梯、岔路、管道+仪表盘、天平、过河踩石、通关闸门……），让结构由画面承载。
- **逐条列出"必须准确出现的标注"**（精确到字），并说清它们的关系/流程/分支/反馈。
- 横版（书是横版页）。

## 铁律与坑（验证过）

- **纯白底**：背景必须 #FFFFFF。生图模型常给 ~#FAFAFA 近白底，挨着书的白页会显出一块灰——`gen_infographics.py` 已内置 `whiten()` 把近白/低饱和像素强制成纯白，仍出灰就调阈值。
- **要上色、但克制**：每张 3~5 种柔和色，用来区分阶段/分支/同类，不要满图彩虹；这是和"纯黑白"的区别，也是和别家风格的区别——保持我们"白底 + 手绘 + 一组书内柔和色"的自有识别度。
- **文字准**：逐张看图（用能读图的工具）验 label，糊字/错字用 `gen_infographics.py --only <name>` 重生成。
- **prompt 别太长**：过度细写每根箭头会让生成超时；写"风格前缀 + 场景隐喻 + 精确标注"即可。
- **政策敏感内容会让模型卡住/拒绝**：把"如何规避/钻空子"类内容重构成"只讲后果、警示"的合规框架。
- 产物是一张**插画图版**，由 `#diagram` 的细线框框住、落在白页上（白底无缝）；不需要色键抠透明。
- 何时用它 vs 功能图：要**有趣/精致/比喻感**就用插画信息图；要**精确可改的技术流程/状态机**用 Mermaid（见 diagrams.md）。
